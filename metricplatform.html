Metric Tracking Platform

Projects' metric performance tracking and dashboarding web app for a top tech company

Interaction Design

Introduction

In the first half of 2020, I worked as the only designer to redesign the major workflow and design new features for a metric performance tracking and dashboarding platform of a top social media company (Platform M for short in the following story). The redesigned workflow and UI based on the latest design system have been launched successfully.

Team & My Role
  • Worked closely with PM and PO and 2 developers on daily basis.
  • Wore multiple hats to define use cases, conduct QA testing, and lead the CSS enhancement in addition to my design work.
Toolbox
  • Figma
  • Design System
  • React / CSS

Challenge

The user experience of the original tool is complex, inefficient, and reactive,  which makes the team hesitant to introduce the tool to larger user groups.
  • The metric adding and dashboard creation process is repetitive
  • The goal setting for metrics has complex logic
  • There is no setting and notification system to notify metric risks
  • Irrelevant projects on the homepage requires users' efforts for searching

I believe I would be the only one who will create a dashboard like this because I am the admin...I need to make sure the tool is easy to use to be adopted by more users.

With this insight from the PO/client, who is also the key persona, the team believes the users' life could be much easier. Thus, I was asked to design for:

How might we create a simplified metric and project management experience that can enhance users' work efficiency and communicate metric performance risk effectively?

Solution Overview

Simplified workflow to manage metrics and view dashboard

Import multiple metrics at a time and view the dashboard with one click on CTA.

Easy-to-follow settings for goal and alert, and proactive metrics tracking

See the results of goal and alert setting immediately based on the input. Get notifications about metric risks from the company's alert system.

Personalized homepage for easy access to users' own projects

Navigate through projects owned, subscribed, and pinnned by the user easily and customize homepage as a pinned project.

Process

As the only designer on the team, I was involved in the whole process from gathering requirements to brainstorming, design, tests, and development.

Due to the time constraints and the natrue of the project, I had limited resource of research and access to end-users. I was able to understand the users' behavior, frustration, and mental modal by having conversations with the PO, who is also the most active user and observing how users set up the metrics and goals on the existing platform. I tested the design with people who have similar technical background with the users to ensure the usability of the design.

Persona

The initial user group mainly comes from one team that focuses on tracking metrics in a specific area. In the future, there might be more users from different teams to track the performance of other types of metrics.

The main persona are engineers/data scientists/program managers who are both editors and viewers of metrics and projects. As a busy user who only spends short amount of time on this tool occasionally, they need the workflow and interface easy to understand and help them get their job done quickly.

User Journey

Design

How might we help users add metrics and creating dashboard efficiently?

The design helps users focus on completing the major use cases efficiently by importing multiple metrics from database at a time and then create a dashboard with one click. Functions that are used less frequently, e.g., add a reference metric (line) is positioned on the metric-level.

How might we make the complex goal and alerts setting easy-to-follow

The design enables users to set multiple quarterly goals for a metric, and then create alerts which will triggered by the metric value's progress toward the goal.

To help the users follow the complex logic and the conditions, the design phrases the conditions like a sentence, e.g. "If the following condition is met...when..., Then crete task..."

The results of the goals and next alert instance will be generated based on users' input as immeidaet feedback so that users can understand easily what the goal / alert is.

How might we make the homepage personalized?

The design of the homepage is providing users a personalized workspace to get access to and view status of projects they own, subscribed, and saved easily.

Users will have the flexibility to set a pinned project as the homepage of the site to help them access the most important project quickly. Many users, like leadership, usually only pay attention to one dashboard that has all children projects connected.

Iteration

I explored different options and iterated on the design based on the feedback from developers, stakeholders, and usability testing results with 5 participants.

Workflow of adding metrics and creating dashboard
  • Option 1: Powerful but complex

    • 😃 Add metrics by clicking on “Add” once

    🙁 Reference metric” is still confusing in the adding flow

    🙁 Efforts to publish connected projects

  • Option 2: Simple and quick

    • 😃 Focus on completing major workflow, adding project-leve metric and viewing dashboard with one button

    😃 Add reference metric on the metric level. Clear about reference metric

    😃 Collapsible section to avoid scroll and help skim

Design alternatives of flow and interaction for importing metrics and creating dashboard

Help users understand the goals setting
  • Option 1 & 2: Use visualization to help users confirm goal settting.

    • 😃 Direct and immediate visual feedback after the setting is done

    🙁 Technical challenges and time constraints in implementing significant UI changes and rendering chart in the modal window.

  • Option 3: Display immediate calculated result in the modal window

    • 😃 Phrase the logic and condition like a narrative

    😃 Display calculated goals and next alert instance based on users' input to support the understanding.

    🙁 Complete the goal setting in the modal window without any visualization support.

Design alternatives for settting goals

Help users understand recurring alerts which apply to all goals
  • Version 1: display alert example at the end.

    • 😃 The alert example is helpful for users to understand the alert setting

    🙁 The example makes users think one alert only applies to one goal.

    🙁 Time condition of "Anytime during the goal period" seems to be vague.

  • Version 2: display example on the side with clear message

    • 😃 User can refer to the alert example easily when setting alerts

    😃 Alert example clearly shows that it is “next instance” of recurring alerts.

    😃 Translate the “goal period” to exact date range in the example.

Impact

The newly designed platform replaced the original tool and  was announced as simplified, cleaner, easier in the release message. The platform became the official tool for the company to track and review their metrics in a specific field.

  • Enhanced work efficiency.

    • Up to 66.7% reduction in clicks to create a dashboard

  • Reduced time to identify risk .

    • Near 0 with alert notification

  • Increasing user base

    • 4.3% increase vs before launch and more to come

With more time and research resource, I'd like to evaluate the task completion rate of goal and alert setting. I am also planning to implement a user satisfaction survey.

Take-aways

With the constraints of limited research resources, I took the hypothesis approach and validated my understanding and design with stakeholders. At the same time, I was trying my best to break these constraints by involving my colleagues to test the design.

I built a collaborative environment by involving the PM and developers, and another designer colleague to brainstorm and critique to avoid design in the silo. As the only designer on the team, I needed to absorb different opinions but at the same time held the designer's point of view to speak for UX.

I enhanced the collaboration between designer and dev by frequently checking in with devs, providing design documents for clear communication, and providing as many details as possible to make CSS enhancement easy.
